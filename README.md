# AskParaAI Backend

Laravel 12 backend for the AskParaAI project.

## Render deployment

This repository now includes a Docker-based Render setup:

- `Dockerfile` builds the PHP app and Vite assets.
- `scripts/start-render.sh` prepares Laravel, runs migrations, and starts the app on Render's assigned `PORT`.
- `render.yaml` provisions the web service and a managed Postgres database.

## Deploy steps

1. Push this repository to GitHub.
2. In Render, create a new Blueprint and point it to this repo.
3. Render will read `render.yaml`, create the web service, and provision `askparaai-db`.
4. In the Render dashboard, set `GEMINI_API_KEY` to your real Gemini API key before the first successful production use.

## Notes

- The Blueprint uses Postgres on Render instead of SQLite so sessions, cache, queues, and app data survive restarts.
- `APP_KEY` is generated by Render through the Blueprint. The start script only creates a temporary fallback if that variable is missing.
- If you do not want automatic migrations during boot, set `RUN_MIGRATIONS=false` in Render.
